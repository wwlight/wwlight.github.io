:::note[Headroom]
[Headroom](https://github.com/headroomlabs-ai/headroom) 是一个本地的上下文优化层：在 LLM 收到之前压缩 tool output、日志、文件、RAG 结果和对话历史，节省 60-95% token，可逆（CCR 机制）。

与 [RTK](https://github.com/rtk-ai/rtk)（Shell 输出裁剪）互补：RTK 处理 shell 层面，Headroom 处理 API 层面。
:::

## 安装

```sh
# 安装 headroom（全功能）
$ uv tool install "headroom-ai[all]"

# 验证
$ headroom --version

```

## 环境变量

```sh
# Proxy 转发目标（上游 LLM API）
export OPENAI_TARGET_API_URL="http://localhost:20128/v1"

```

`OPENAI_TARGET_API_URL` 告诉 headroom proxy 将压缩后的请求转发到哪个上游。

## 启动 Proxy

```sh
# 手动启动（前台，launchd 注册后无需手动）
$ OPENAI_TARGET_API_URL=http://localhost:20128/v1 headroom proxy --port 8787
```

## 服务管理

```sh
# 注册并启动（开机自启）
$ launchctl load ~/Library/LaunchAgents/headroom.proxy.plist

# 手动停止
$ launchctl unload ~/Library/LaunchAgents/headroom.proxy.plist

# 查看状态（第二列为 0 表示正常）
$ launchctl list | grep headroom
```

Plist 文件内容（`~/Library/LaunchAgents/headroom.proxy.plist`）：
```xml
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN" "http://www.apple.com/DTDs/PropertyList-1.0.dtd">
<plist version="1.0">
<dict>
  <key>Label</key>
  <string>headroom.proxy</string>
  <key>ProgramArguments</key>
  <array>
    <string>$HOME/.local/bin/headroom</string>
    <string>proxy</string>
    <string>--port</string>
    <string>8787</string>
  </array>
  <key>EnvironmentVariables</key>
  <dict>
    <key>OPENAI_TARGET_API_URL</key>
    <string>http://localhost:20128/v1</string>
  </dict>
  <key>RunAtLoad</key>
  <true/>
  <key>KeepAlive</key>
  <true/>
  <key>StandardOutPath</key>
    <string>$HOME/.headroom/logs/launchd-stdout.log</string>
  <key>StandardErrorPath</key>
  <string>$HOME/.headroom/logs/launchd-stderr.log</string>
</dict>
</plist>
```

:::note[可移植性]
launchd plist 不支持 `~`、`$HOME` 等变量展开，所有路径须为绝对路径。使用前将 `$HOME` 替换为实际路径（`echo $HOME`）。
:::

## 集成到 OpenCode

### 方式 A：一键集成（通用 OpenAI/Anthropic）

```sh
$ headroom wrap opencode
```

自动修改 `opencode.json`、启动 proxy、注册 MCP、注入 AGENTS.md。

### 方式 B：自定义上游（如 omniroute）

当上游不是官方 API，而是自定义 OpenAI 兼容代理时，手动配置：

```json
{
  "mcp": {
    "headroom": {
      "type": "local",
      "command": ["headroom", "mcp", "serve", "--proxy-url", "http://localhost:8787"],
      "enabled": true
    }
  },
  "provider": {
    "omniroute": {
      "api": "openai-compatible",
      "options": {
        "baseURL": "http://127.0.0.1:8787/v1"
      }
    }
  }
}
```

provider baseURL 指向 headroom proxy，proxy 通过 `OPENAI_TARGET_API_URL` 转发到真实上游。

## 验证

```sh
# 检查 proxy 状态
$ curl http://localhost:8787/health

# 查看压缩统计（先进行一次对话）
$ curl http://localhost:8787/stats

# 健康检查
$ headroom doctor
```

## 卸载

```sh
# 恢复 opencode.json + 注销 MCP + 移除 AGENTS.md 指令
$ headroom unwrap opencode

# 彻底卸载 headroom
$ uv tool uninstall headroom-ai
```

## 常用命令

| 操作 | 命令 |
|------|------|
| 启动 proxy（前台） | `OPENAI_TARGET_API_URL=http://... host headroom proxy --port 8787` |
| 注册为系统服务 | `launchctl load ~/Library/LaunchAgents/headroom.proxy.plist` |
| 停止服务 | `launchctl unload ~/Library/LaunchAgents/headroom.proxy.plist` |
| 查看服务状态 | `launchctl list \| grep headroom` |
| 一键集成 OpenCode | `headroom wrap opencode` |
| 查看统计 | `curl http://localhost:8787/stats` |
| 查看仪表盘 | `curl http://localhost:8787/dashboard` |
| 健康检查 | `headroom doctor` |
| 恢复原始配置 | `headroom unwrap opencode` |

## 预期节省

| 场景 | 节省率 |
|------|--------|
| 代码搜索 | 70-92% |
| grep / 日志分析 | 60-80% |
| 代码库探索 | 40-60% |
| 日常编码 | 30-50% |

## 注意事项

- 需要至少一个 LLM API key（Anthropic / OpenAI / DeepSeek）
- Headroom 对模型完全透明，不限 provider
- 自定义上游需设置 `OPENAI_TARGET_API_URL`，否则 proxy 默认转发到官方 API
- `uv tool install` 自动管理 Python 版本，无需手动升级
- launchd 注册后 proxy 后台常驻、开机自启，不依赖终端会话
