:::note[Docker]
[Docker](https://docker.com) 是一个用于开发、交付和运行容器化应用的开放平台，帮助你将应用与基础设施分离，实现快速交付。适用于本地开发环境统一、微服务部署、CI/CD 流水线、多服务编排（docker-compose）、快速原型验证等场景。
:::

:::note[Colima]
[Colima](https://github.com/abiosoft/colima) 是 macOS（及 Linux）上的容器运行时，以最小配置即可运行 Docker 和 Containerd，支持 Intel 和 Apple Silicon。适用于替代 Docker Desktop、本地容器开发、多实例管理、Kubernetes 集群（--kubernetes）、Containerd 运行时、GPU 加速 AI 容器（--vm-type krunkit）、Rosetta 2 模拟（--vz-rosetta）等场景。
:::

## 安装步骤

<Steps>
1. 安装 Docker、Docker Compose 及 Colima

    ```sh
    $ brew install docker docker-compose colima
    ```

2.  启动 Colima 服务

    ```sh
    $ brew services start colima
    ```

3.  验证安装

    ```sh
    $ docker info
    ```

</Steps>

## [Docker 命令大全](https://docs.docker.com/reference/cli/docker/)

### 镜像管理

```sh {1,2,6}
$ docker images                                                           # 列出本地镜像
$ docker pull <name>[:tag]                                                # 拉取镜像
$ docker push <name>[:tag]                                                # 推送镜像
$ docker rmi <image>                                                      # 删除镜像
$ docker rmi $(docker images -q)                                          # 删除所有镜像
$ docker build -t <name> .                                                # 从 Dockerfile 构建镜像
$ docker tag <src> <target>                                               # 给镜像打标签
$ docker search <keyword>                                                 # 搜索镜像
```

### 容器管理

```sh {1,3,5,12,14}
$ docker ps                                                               # 列出运行中的容器
$ docker ps -a                                                            # 列出所有容器（含已停止）
$ docker run <image>                                                      # 创建并启动容器
$ docker run -d <image>                                                   # 后台运行
$ docker run -it <image> /bin/bash                                        # 交互模式
$ docker run -p 8080:80 <image>                                           # 端口映射
$ docker run -v /host:/container <image>                                  # 挂载卷
$ docker run --name <name> <image>                                        # 指定容器名
$ docker start <container>                                                # 启动容器
$ docker stop <container>                                                 # 停止容器
$ docker restart <container>                                              # 重启容器
$ docker rm <container>                                                   # 删除容器
$ docker rm $(docker ps -aq)                                              # 删除所有容器
$ docker logs <container>                                                 # 查看日志
$ docker logs -f <container>                                              # 实时跟踪日志
$ docker exec -it <container> /bin/bash                                   # 进入容器
$ docker cp <src> <container>:<dest>                                      # 复制文件到容器
$ docker cp <container>:<src> <dest>                                      # 从容器复制文件
$ docker stats                                                            # 查看容器资源占用
$ docker top <container>                                                  # 查看容器进程
$ docker inspect <container>                                              # 查看容器详细信息
$ docker rename <old> <new>                                               # 重命名容器
$ docker container prune                                                  # 清理所有已停止容器
```

### Docker Compose

```sh {2}
$ docker compose up                                                       # 启动所有服务
$ docker compose up -d                                                    # 后台启动
$ docker compose down                                                     # 停止并删除所有服务
$ docker compose down -v                                                  # 停止并删除服务及卷
$ docker compose ps                                                       # 列出服务状态
$ docker compose logs                                                     # 查看所有服务日志
$ docker compose logs -f                                                  # 实时跟踪日志
$ docker compose restart                                                  # 重启所有服务
$ docker compose stop                                                     # 停止所有服务
$ docker compose start                                                    # 启动所有服务
$ docker compose build                                                    # 重新构建镜像
$ docker compose pull                                                     # 拉取最新镜像
$ docker compose exec <service> /bin/bash                                 # 进入服务容器
```

### 系统管理

```sh {6,8,11}
$ docker info                                                             # 查看系统信息
$ docker version                                                          # 查看版本
$ docker system df                                                        # 查看磁盘占用
$ docker system prune                                                     # 清理所有未使用的资源
$ docker system prune -a                                                  # 清理所有未使用的资源（含镜像）
$ docker system prune --volumes                                           # 清理所有未使用的资源（含数据卷）
$ docker system events                                                    # 实时查看 Docker 守护进程事件
$ docker system events --since 1h                                         # 查看过去 1 小时的事件
$ docker system events --filter 'type=container' --filter 'event=start'   # 过滤容器启动事件
```

### 镜像进阶操作

```sh {4,5,9}
$ docker history <image>                                                  # 查看镜像分层历史
$ docker image prune                                                      # 清理悬空（dangling）镜像
$ docker image prune -a                                                   # 清理所有未被容器引用的镜像
$ docker image prune -a --filter "until=24h"                              # 清理 24 小时前的未使用镜像
$ docker image prune --filter "label=deprecated"                          # 清理带特定标签的镜像
$ docker builder prune                                                    # 清理构建缓存
$ docker builder prune -a                                                 # 清理所有构建缓存
$ docker builder prune --keep-storage 2GB                                 # 保留 2GB 构建缓存
```

### 网络管理

```sh
$ docker network ls                                                       # 列出网络
$ docker network create <name>                                            # 创建网络
$ docker network rm <name>                                                # 删除网络
$ docker network connect <net> <container>                                # 容器接入网络
$ docker network disconnect <net> <container>                             # 容器断开网络
$ docker network inspect <name>                                           # 查看网络详情
```

### 数据卷管理

```sh
$ docker volume ls                                                        # 列出数据卷
$ docker volume create <name>                                             # 创建数据卷
$ docker volume rm <name>                                                 # 删除数据卷
$ docker volume prune                                                     # 清理未使用的数据卷
$ docker volume inspect <name>                                            # 查看数据卷详情
```

### 备份与迁移

```sh {1,3,5,8,11}
$ docker commit <container> <name>:<tag>                                  # 将容器保存为新镜像（环境快照）
$ docker export -o output.tar <container>                                 # 导出容器文件系统为 tar 归档（不含卷）
$ docker import output.tar <name>:<tag>                                   # 从 tar 归档导入为镜像
$ docker save -o output.tar <image>                                       # 导出镜像为 tar（离线迁移）
$ docker load -i output.tar                                               # 从 tar 加载镜像
$ docker tag <src> <target>                                               # 复刻/重命名镜像
```

### 数据卷备份与恢复

```sh {2,3}
# 备份卷到宿主机
$ docker run --rm -v <volume>:/data -v $(pwd):/backup alpine tar czf /backup/vol-backup.tar.gz -C /data .
# 从宿主机恢复卷
$ docker run --rm -v <volume>:/data -v $(pwd):/backup alpine tar xzf /backup/vol-backup.tar.gz -C /data
# 跨主机迁移卷
$ docker run --rm -v <volume>:/data alpine tar czf - -C /data . | ssh user@host "docker run --rm -i -v <volume>:/data alpine tar xzf - -C /data"
```

### 调试与诊断

```sh {1,7,9,10}
$ docker diff <container>                                                 # 查看容器文件变更（跟踪改了啥）
$ docker port <container>                                                 # 查看容器端口映射
$ docker port <container> 80/tcp                                          # 查看指定协议的端口映射
$ docker logs --since 30m <container>                                     # 查看最近 30 分钟日志
$ docker logs --tail 100 -f <container>                                   # 跟踪最后 100 行日志
$ docker logs --until 10m <container>                                     # 查看 10 分钟前的日志
$ docker inspect --format '{{.NetworkSettings.IPAddress}}' <container>    # 查容器 IP
$ docker inspect --format '{{range .NetworkSettings.Networks}}{{.IPAddress}}{{end}}' <container>  # 查所有网络 IP
$ docker cp <src> <container>:<dest>                                      # 复制文件到容器
$ docker cp <container>:<src> <dest>                                      # 从容器复制文件
```

## Colima 常用命令

```sh
$ colima status                                                           # 查看状态
$ colima start                                                            # 启动
$ colima stop                                                             # 停止
$ colima ssh                                                              # SSH 进入虚拟机，exit 或 Ctrl-D 退出
$ colima list                                                             # 列出实例
$ colima delete                                                           # 删除虚拟机
$ colima version                                                          # 查看版本
$ colima nerdctl install                                                  # 安装 nerdctl 别名
```
Colima 的 CPU、内存、磁盘等配置通过 `~/.colima/default/colima.yaml` 管理。
